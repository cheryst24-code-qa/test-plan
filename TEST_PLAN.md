## 2. Введение
- **Цель:** Проверить стабильность и соответствие требованиям новых функций: редактируемые комментарии, загрузка файлов до 50MB, @упоминания пользователей и настройки уведомлений.
- **Аудитория:** QA-команда, Frontend/Backend разработчики, Product Owner, Tech Lead.
- **Определения:** 
  - `Smoke` – проверка критического пути (создание задачи → комментарий → загрузка файла)
  - `Blocker` – дефект Severity 1, делающий тестирование невозможным
  - `Pass Rate` – доля успешно пройденных тест-кейсов из общего числа

## 4. Подход к тестированию
| Аспект               | Описание                                                                 |
|----------------------|--------------------------------------------------------------------------|
| **Уровни**           | Unit (Dev) → Integration → System → Acceptance (UAT)                     |
| **Типы**             | Функциональные, регресс, smoke, exploratory, UX-проверки                 |
| **Техники дизайна**  | Граничные значения (размер файла 0/50/51MB), эквивалентные классы, State Transition для статусов файлов |
| **Автоматизация**    | `Playwright` + `Allure`. Покрытие критического пути: `68%`. Запуск в GitHub Actions |
| **Баг-менеджмент**   | Jira. Workflow: `Open → In Progress → Ready for QA → Verified → Closed` |
