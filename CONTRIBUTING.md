# CONTRIBUTING.md
Документ описывает стандарты оформления дефектов для проекта **TaskFlow**.

## Как оформить баг-репорт
Создавайте Issues только после проверки на `staging.taskflow.app` и поиска дублей.

### Обязательные поля
| Поле | Требование | Пример |
|------|------------|--------|
| **Title** | `[Module] Краткое описание проблемы` | `[Attachments] Загрузка PDF 51MB блокирует UI без сообщения об ошибке` |
| **Environment** | ОС, браузер, версия сборки, env | `macOS 14.4, Chrome 124.0.6367, Build v2.0.0-rc1, Staging` |
| **Steps to Reproduce** | Нумерованный список, без лишних действий | `1. Открыть задачу TF-405<br>2. Нажать "Upload"<br>3. Выбрать файл test_51mb.pdf` |
| **Expected Result** | Что должно произойти по требованиям | `Отображение ошибки "Максимальный размер 50MB", UI остаётся активным` |
| **Actual Result** | Что произошло фактически + скриншот/лог | `Белый экран на 15 сек, кнопка Upload недоступна, в консоли: `Error: PayloadTooLarge`` |
| **Severity / Priority** | По матрице ниже | `Severity: High, Priority: Medium` |
| **Attachments** | Логи, HAR-файл, видео, скриншоты | `console.log, network_har_51mb.zip` |

### Матрица Severity vs Priority
| Severity | Priority | Когда применять |
|----------|----------|-----------------|
| **Critical** | Immediate | Потеря данных, падение сервиса, блокировка основного потока (создание/редактирование задачи) |
| **High** | High | Ключевая функция работает некорректно, обходной путь сложен (файлы не скачиваются) |
| **Medium** | Medium | Отклонение от требований, обходной путь существует (некорректное превью JPG) |
| **Low/Cosmetic** | Low | Косметика, UX-неточности, опечатки в тултипах |

## Процесс работы
1. **QA**: создаёт Issue → назначает `Assignee: @frontend-dev` → вешает лейблы `bug`, `severity/high`, `priority/medium`
2. **Dev**: переводит в `In Progress` → фиксирует → переводит в `Ready for QA`
3. **QA**: проверяет на staging → если OK → `Verified` → `Closed`. Если NOK → `Reopened` с комментарием и новой сборкой.

## Шаблон Issue
Файл `.github/ISSUE_TEMPLATE/bug_report.md` уже содержит эту структуру. При создании Issue GitHub подставит её автоматически.
